# designPatterns

https://refactoring.guru

1.Singleton — «один объект на всю систему»

2.Factory Method — «подкласс решает, что создавать»

3.Abstract Factory — «создать совместимый набор объектов»

4.Builder — «собирать объект шаг за шагом»

5.Adapter — «сделать несовместимое совместимым»

6.Decorator — «добавить поведение, не меняя класс»

7.Facade — «спрятать сложность за простым API»

8.Strategy — «менять алгоритм во время выполнения»

9.Template Method — «алгоритм общий, шаги разные»

10.Iterator — «проходить по элементам коллекции, не раскрывая её внутреннюю структуру»
